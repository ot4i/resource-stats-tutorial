Tracking Resource Statistics

Learn how turn on resource statistics tracking and view them in the WebUI.
OVERVIEW

This tutorial shows how a user would go about enabling resource statistics tracking and then monitor said statistics in real-time. The example uses a pair of message flows which implement transformations and file input/output as the core demonstrative operations.

The server.conf.yaml file will be introduced as a useful way for tweaking the behaviour of an integration server. In this example we will only enable resource statistics tracking.

The user will then make several requests to the server and then shown the resulting raw and processed files in the server working directory.

Finally, they will be guided through accessing the WebUI of the server and how to use it to view various resource statistics. More specifically, the statistics regarding File transferring, processing and output.

This tutorial takes approximately 15 minutes.
Tutorial Topics
  • Finding and editing the server config file
  • Manually enabling resource statistics tracking
  • Configuring File In/Output nodes within flows
  • Accessing the WebUI
  • Monitoring the resource statistics in real-time
Learning outcomes
  • Understand how to locate the server config file, turn on resource statistics tracking and monitor them in real time using the WebUI.
